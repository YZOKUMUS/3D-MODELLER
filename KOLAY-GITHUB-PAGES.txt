GITHUB PAGES — 404 cozumu (acele)

A) NEDEN 404?
   1) GitHub Actions hesabinizda "billing" kilidi varsa OTOMATIK workflow CALISMAZ.
      Actions sekmesinde kirmizi hata: "account is locked due to a billing issue"
      => https://github.com/settings/billing adresinden kontrol edin.
   2) Ucretsiz hesapta repo GIZLI (private) ise github.io sayfasi genelde acilmaz.
      Repo: Settings -> Danger Zone -> Change visibility -> PUBLIC yapin.

B) EN KOLAY CALISAN YOL (Actions gerekmez)

   Otomatik: main'e kod push'ladiginda "docs otomatik web export" workflow'u
   docs/ klasorunu guncelleyip pushlar (repo -> Actions). Pages yine branch /docs.

   1) GitHub: https://github.com/YZOKUMUS/3D-MODELLER/settings/pages
      "Build and deployment" -> Source: "Deploy from a branch"
      Branch: main   Folder: /docs   Save

   2) Bilgisayarda (model-market klasorunde):
      npm run publish:docs
      (biraz surer, cok sayida resim var)
      Not: export sonrasi dist/404.html otomatik; sayfa yenileme / model linki GitHub'da duzgun acilir.
      Masaustu kisayolu tarayiciyi http://localhost:8082/3D-MODELLER/ acar (GitHub ile ayni base yol).
      Ana ekrana eklenen PWA: ilk kez bu manifest ile yayindan sonra kisayolu silip
      siteden tekrar "Ana ekrana ekle" dene; her yeni deploy farkli ?build= ile guncellenir.

   3) Yukle:
      git add docs
      git commit -m "docs: web sitesi"
      git push origin main

   4) 2-5 dakika bekleyin, sonra acin:
      https://YZOKUMUS.github.io/3D-MODELLER/

C) (Istege bagli) GitHub Actions ile yayin
   Repo -> Actions -> "GitHub Pages web yayini" -> "Run workflow" (manuel).
   Hesapta billing/Actions sorunu varsa bu da calismayabilir; o zaman B maddesi yeterli.

D) Hala istemezseniz
   Netlify veya Cloudflare Pages: ucretsiz, surukle-birak veya repo bagla (ayri anlatilir).
